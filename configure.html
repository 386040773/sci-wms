Adding datasets to the server

Administration Site

http://server:port/admin

In order to add and remove datasets users must log in to the sci-wms admin page. This site allows administrators to create and remove users, update passwords, add server/service level metadata that will show up in getCapabilities documents, and configure datasets.

The default username is: sciwmsuser

The default password is: sciwmspassword

At the very least, the default sciwmsuser password should be changed when users first log in while setting up the new server.

Depending on the version of Django you have installed you may have a problem logging into the admin site, even with the correct password. An easy way solve this problem is to simply to run the following command to reset the sciwmsuser password from the command line.

python manage.py changepassword sciwmsuser

Adding a new dataset

Select the Datasets link from the main admin page. Choose the add new dataset option. Fill in the required fields. The name will be a unique dataset id. The path can either be a local file path or a valid OpenDAP link (provided that opendap is enabled in the netcdf lib). Title and abstract parameters are used as human readable metadata available in the getCapabilities document for the dataset.

Enable automatic updating if sci-wms should check for updates to the source dataset periodically (like in the case of a forecast aggreation).

The optional parameters are only required if you would like to be able to quickly test the getMap services in the embeded openlayers test page.

Once the required fields have been populated, it is important to select save. This ensures that the changes you have made will be pushed into the sci-wms database.
